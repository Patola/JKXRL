# v0.3 Linux-Only Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove the Windows and Android build targets from JKXRL, relocate the engine source out of the leftover Android NDK layout, and tag a clean Linux-only `v0.3`.

**Architecture:** Mechanical repo restructuring. The engine source moves `Projects/Android/jni/OpenJK/` → `OpenJK/` (history preserved via `git mv`). Windows/Android scaffolding is deleted; the weapon pk3 game assets are relocated first so they survive. The proven Linux build is the safety net after each task. Key insight lowering risk: on Linux every `if(WIN32)`/`if(MSVC)` branch in CMake evaluates false, so deleting Windows files cannot break the Linux build — the CMake dead-branch cleanup is purely cosmetic and lives in its own revertible task.

**Tech Stack:** CMake (GCC/Clang), OpenXR (X11/GLX), SDL2, OpenGL, makepkg (Arch PKGBUILD), POSIX shell.

**Spec:** `docs/superpowers/specs/2026-06-30-v03-linux-only-cleanup-design.md`

## Global Constraints

- **No behavior changes:** engine, renderer, shadows, VR, gameplay are untouched. Only build/target structure changes.
- **Build flags unchanged:** Release with `-O1 -fno-strict-aliasing`, LTO disabled (already set in CMake + PKGBUILD `options=('!lto')`). Do not alter optimization.
- **`rd-gles` is kept** as dead reference (not compiled, not deleted).
- **No pushing:** local commits + annotated tag only. Never `git push` unless explicitly asked.
- **Verification bar:** `./build_linux.sh` green AND `makepkg` produces a valid package. Full VR gameplay is confirmed by the user (this environment is headless).
- **Preserve history:** use `git mv` for the source relocation.
- Working directory for all relative paths: repo root `/home/patola/workspace/opencode/JKXRL`.
